AyurVAID

Premium AI-Powered Ayurvedic Health Intelligence System — A comprehensive full-stack application that combines Machine Learning, NLP, Retrieval-Augmented Generation, and Explainable AI to deliver personalised Ayurvedic wellness guidance.

Application Overview

AyurVAID is a Next.js + Express full-stack web application that enables users to discover their Ayurvedic body constitution (Dosha), receive ML-classified health insights with SHAP-based explainability, and interact with a hybrid AI chatbot for personalised wellness recommendations.

How It Works — End-to-End Flow

1

User Authentication

Users register or log in via Firebase Authentication. The server issues a JWT token that is attached to all subsequent API requests. The middleware/auth.js middleware validates tokens on protected routes.

2

Dosha Assessment (ML Classification)

The user completes 25 physiological and behavioural questions mapped to CSV columns from a 5,000-row Prakriti dataset. The answers are sent to POST /api/dosha/analyze, which calls DoshaAnalyzer.js. This spawns a Python child process running catboost_model.py — a trained CatBoost gradient-boosting classifier. The model returns probability scores per Dosha, SHAP feature attributions, and constitution type. The ExplainableAI engine then enriches the result with human-readable reasoning. Finally, the AI Service Manager generates a "Wisdom Summary" using the active LLM provider.

3

Health Chatbot (Hybrid AI + RAG)

Users send messages via POST /api/chat/message. The AIServiceManager orchestrates the active provider. Before dispatching to any AI, a RAG step searches the KnowledgeBase (450+ herbs, 120+ principles, 80+ medicines from JSON datasets) using weighted keyword scoring with Dosha-specific boosting. Retrieved context is injected into the system prompt. The active AI provider then generates a personalised response. The ExplainableAI engine wraps the response with transparent reasoning and source citations.

4

Provider Fallback Chain

If the active provider fails, the AIServiceManager automatically falls back to the CustomAI rule-based engine, which is always available. The Rasa hybrid provider also has an internal fallback—if Rasa NLU returns its default response or is offline, it hands off to Gemini 2.0 Flash as a generative fallback.

Component 1 — CatBoost ML Classifier + SHAP xAI

Classification & Explainability

A gradient-boosted decision tree model trained on a 5,000-row Prakriti dataset. It classifies users into Vata, Pitta, or Kapha dosha types from 25 encoded features. SHAP (SHapley Additive exPlanations) values are computed natively via CatBoost's get_feature_importance(type='ShapValues') to identify which user traits most influenced the prediction.

Component 2 — Hybrid Rasa NLU + Gemini Generative Fallback

NLP + Generative AI

Rasa NLU uses a DIET (Dual Intent and Entity Transformer) classifier with character-level n-gram featurisers for intent classification of Ayurvedic queries. A FallbackClassifier (threshold 0.3) detects ambiguous queries. When Rasa is ambiguous, offline, or returns an empty response, the system seamlessly hands control to Google Gemini 2.0 Flash as a generative LLM fallback with dosha-contextualised system prompts.

Component 3 — RAG (Retrieval-Augmented Generation) Pipeline

Knowledge Retrieval

Before any AI provider generates a response, the system performs a Retrieval-Augmented Generation step. The KnowledgeBase.js service loads herbs, Ayurvedic principles, and medicines from JSON datasets. A weighted keyword search with Dosha-specific disambiguation scoring retrieves the most relevant entries. These are formatted and injected into the system prompt as verified classical knowledge.

Component 4 — CustomAI Rule-Based Engine

Deterministic / Always Available

A fully offline, rule-based AI that serves as the system's ultimate fallback. It uses keyword-based intent classification across 7 health categories (food, lifestyle, exercise, health, mental, sleep, digestion), extract contextual signals (urgency, severity, time-of-day, season, symptoms), and generates dosha-personalised responses from the Ayurvedic knowledge base. It also invokes the RAG KnowledgeBase for expert herb/principle data.

Component 5 — Explainable AI (xAI) Engine

Transparency & Trust

The ExplainableAI.js module provides interpretable explanations for every AI decision. For dosha analysis, it narrates why the CatBoost model chose a specific classification, referencing SHAP top-contributors and counter-indicators. For chatbot recommendations, it generates methodology, confidence, and source attribution metadata. This ensures every AI output is transparent and auditable.

Component 6 — AI Service Manager (Orchestrator)

Multi-Provider Orchestration

The central nervous system of the AI layer. AIServiceManager.js provides a unified interface to 5 AI providers (OpenAI, LocalAI/Ollama, HuggingFace, CustomAI, Rasa). It handles provider initialisation, availability checks, automatic fallback to CustomAI, and embeds the RAG retrieval step before every generation call.
