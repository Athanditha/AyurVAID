Machine Learning

1. CatBoost Dosha Prediction and SHAP Explanation

Shows the step-by-step message flow from the user submitting their questionnaire answers, through the backend spawning a Python prediction script, to receiving dosha scores and SHAP explanations.

sequenceDiagram
    actor User
    participant App as Web Application
    participant Analyzer as ML Pipeline
    participant Model as CatBoost & SHAP

    User->>App: Submits health questions
    activate App
    App->>Analyzer: Forward answers for analysis
    activate Analyzer

    Analyzer->>Model: Predict dosha & compute SHAP importances
    activate Model
    Model-->>Analyzer: Returns dosha scores & key feature insights
    deactivate Model

    Analyzer-->>App: Returns unified analysis result object
    deactivate Analyzer
    App-->>User: Displays results with visual indicators
    deactivate App
Key files: catboost_model.py, DoshaAnalyzer.js, routes/dosha.js
Note: The Python script communicates with Node.js by printing JSON to standard output, which the backend reads and parses.
AI Orchestration

2. AI Service Manager - Orchestrator

Shows how the AI Service Manager receives a user message, enriches it with knowledge base context, routes it to the active AI provider, and handles fallback when the primary provider fails.

sequenceDiagram
    actor User
    participant App as Web Application
    participant Manager as AI Orchestrator
    participant KB as Knowledge Base
    participant Provider as AI Provider (e.g. Gemini)

    User->>App: Sends a health question
    App->>Manager: Route message
    activate Manager

    Note over Manager,KB: RAG Step - Enrich with knowledge
    Manager->>KB: Retrieve relevant Ayurvedic context
    activate KB
    KB-->>Manager: Return matching herbs & principles
    deactivate KB

    Manager->>Manager: Inject context into prompt

    Manager->>Provider: Forward prompt to active Provider
    activate Provider
    Provider-->>Manager: Return AI-generated response
    deactivate Provider
    
    Manager-->>App: Forward finalized response (with Provider badge)
    deactivate Manager
    
    App-->>User: Display the AI response in chat
Key files: AIServiceManager.js, routes/chat.js
Note: The fallback is automatic and invisible to the user. The response includes a provider badge so the user can see which AI answered.
AI Provider

3. Gemini LLM Provider

Shows how the Gemini provider builds a personalised Ayurvedic prompt, prepares conversation history, sends it to Google's Gemini API, and retries automatically if the service is temporarily busy.

sequenceDiagram
    participant Manager as AI Orchestrator
    participant Gemini as Gemini Provider
    participant API as External LLM API

    Manager->>Gemini: Request response generation
    activate Gemini

    Gemini->>Gemini: Build customized system instructions
    Gemini->>Gemini: Inject retrieved RAG context & Dosha Profile
    Gemini->>Gemini: Append sanitised conversation history

    Gemini->>API: Send engineered prompt payload
    activate API
    API-->>Gemini: Return generated text
    deactivate API

    Gemini->>Gemini: Format response with tracking stats
    Gemini-->>Manager: Return output string
    deactivate Gemini
Key file: GeminiAI.js
Note: The wait times double each retry (2s, 4s, 8s) — this is called exponential backoff and prevents overloading a busy API.
AI Provider

4. Rule-Based AI Provider

Shows how the rule-based engine classifies the user's question by topic, picks a dosha-specific response template, and enriches the reply with expert knowledge from the database.

sequenceDiagram
    participant Manager as AI Orchestrator
    participant RuleAI as Rule-Based Engine
    participant KB as Knowledge Base

    Manager->>RuleAI: Request response generation
    activate RuleAI

    RuleAI->>RuleAI: Determine subject intent & extract core topic
    RuleAI->>RuleAI: Extract referenced Dosha from context
    
    RuleAI->>KB: Search best guidelines/recipes for Topic + Dosha
    activate KB
    KB-->>RuleAI: Return static curated recommendations
    deactivate KB

    RuleAI->>RuleAI: Construct response utilizing static templating
    RuleAI-->>Manager: Reply indicating localized processing
    deactivate RuleAI
Key file: CustomAI.js
Note: This provider runs entirely locally with no API calls, making it always available as a guaranteed fallback.
Data Pipeline

5. RAG Knowledge Retrieval

Shows how the system searches the knowledge base before generating a response, then injects relevant Ayurvedic facts into the AI's instructions so the answer is grounded in verified information.

sequenceDiagram
    participant Manager as AI Orchestrator
    participant KB as In-Memory Database
    participant API as External Generation

    Manager->>KB: Search documents via keywords extraction
    activate KB
    KB->>KB: Parse logic & compute weighting factor
    KB-->>Manager: Resolve with top 3 domain nodes
    deactivate KB

    Manager->>Manager: Intertwine extracted facts to prompt
    Manager->>API: Generate payload request including contexts
    activate API
    API-->>Manager: Respond matching RAG criteria constraint
    deactivate API
Key files: KnowledgeBase.js, AIServiceManager.js
RAG means "look up facts first, then generate." This ensures the AI cites real herbs and principles rather than making things up.
Data Layer

6. Knowledge Base Search

Shows inside the search engine: how a question is broken into keywords, scored against every item in the database, boosted for dosha relevance, and filtered down to the best matches.

sequenceDiagram
    participant Caller as Orchestrator Component
    participant KB as NLP Knowledge Engine

    Caller->>KB: Request query vectorization
    activate KB

    KB->>KB: Strip language invariants & generate tokens
    KB->>KB: Assess textual keyword relevance metric (+2)
    KB->>KB: Assess rigid entity naming metric (+10)
    KB->>KB: Assign specific dosha balancing bias weights

    KB->>KB: Sort array descending iteratively
    KB-->>Caller: Terminate querying and yield dataset
    deactivate KB
Key file: KnowledgeBase.js
Scoring: Text match = +2, Name match = +5, Exact name = +10, Balances queried dosha = +10, Mentions dosha = +3, Unrelated = -5. Threshold is 5 points.
