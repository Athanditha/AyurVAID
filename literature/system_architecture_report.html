AyurVAID Architecture

Technical Blueprint of the Ayurvedic RAG Engine

Knowledge Layer

1. Structured Knowledge Base

The foundation of the system is a local, high-fidelity library of Ayurvedic wisdom, extracted from peer-reviewed datasets and classical texts.

Assets:
  • herbs.json: 704 records (Rasa, Virya, Vipaka)
  • principles.json: 162 Ayurvedic concepts
  • medicines.json: Therapeutic dosages and uses
Mechanism: Data is stored in high-density JSON format for O(1) retrieval speed during system initialization and optimized memory usage.
Retrieval Layer

2. Keyword-Based RAG Engine

A custom-built Retrieval-Augmented Generation (RAG) system that operates entirely locally to ensure 100% uptime and zero API cost.

Core Component: KnowledgeBase.js
Scoring Algorithm: Multi-factor Keyword Scoring with Dosha-Specific Boosting. If a user query mentions a Dosha, the system prioritizes herbs that balance that specific state.
Intelligence Layer

3. AI Service Manager (Orchestrator)

A unified interface that manages multiple AI providers and handles the "Augmentation" step of RAG.

Augmentation Process: Intercepts user messages, performs a Knowledge Base search, and "stuffs" the retrieved snippets into the AI's system prompt before generation.
Hybrid Fallback: Seamlessly switches between Gemini (Cloud), Rasa (NLP), and CustomAI (Local Rules) based on API availability and quota status.
Execution Layer

4. CustomAI (Local Expert)

A rule-based intelligence engine designed to provide expert advice when cloud models are throttled or offline.

Personalization: Detects Doshas (Vata, Pitta, Kapha) in real-time from conversation history or assessment profiles.
Synthesis: Combines Ayurvedic reasoning templates with the dynamic data retrieved via RAG to create a human-like, data-grounded response.
Frontend Layer

5. Next.js Glassmorphic UI

A premium, reactive interface that focuses on Explainable AI (xAI) and user engagement.

Explainable AI: Collapsible analysis cards show the "Why" behind every bot response, citing traditional sources and confidence scores.
State Management: Real-time sidebar synchronization ensures that new consultations are visible instantly without page reloads.
