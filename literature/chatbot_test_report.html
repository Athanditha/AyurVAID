🔧 Fixes Applied
01
Gemini Model Name Updated to gemini-2.0-flash Critical Fix
The gemini-1.5-flash model name was returning a 404 error from the Google Generative AI API under the v1beta endpoint. After checking the Google API, the model was updated to the stable gemini-2.0-flash which resolves the 404 and also cascades to fix the 3 dosha keyword content warnings (Vata, Pitta, Kapha queries now return proper Gemini-generated responses instead of the static "network issues" error message).
📄 server/services/RasaAI.js
// Line 80 — RasaAI.generativeFallback() - const model = this.genAI.getGenerativeModel({ model: "gemini-1.5-flash", ... }) + const model = this.genAI.getGenerativeModel({ model: "gemini-2.0-flash", ... })
02
Gemini Model Updated in Test Suite Direct Calls Test Fix
The test suite itself also used gemini-1.5-flash in the direct Gemini API test (Suite 5). Updated the test to use gemini-2.0-flash to ensure the test correctly validates API connectivity.
📄 server/test_chatbot.js
// Suite 5 — Gemini API Direct Call test - const model = genAI.getGenerativeModel({ model: 'gemini-1.5-flash', ... }) + const model = genAI.getGenerativeModel({ model: 'gemini-2.0-flash', ... })
03
Added provider Field to RasaAI Fallback Response Traceability Fix
The generativeFallback() method in RasaAI.js was setting provider: 'hybrid-rasa-gemini' on the response, but AIServiceManager.generateResponse() was overwriting the entire return value with provider: this.currentProvider ("rasa"), masking the true provider chain. Added preservation of the inner provider detail in AIServiceManager as a innerProvider field so the original provider attribution is not lost.
📄 server/services/AIServiceManager.js
// generateResponse() return value — preserve inner provider return { ...response, provider: this.currentProvider, + innerProvider: response.provider, // preserve hybrid-rasa-gemini etc. timestamp: new Date().toISOString() };
04
Added gemini-2.0-flash Fallback in test_hybrid_ai.js Legacy Test Fix
The existing test_hybrid_ai.js file (TEST 4: Gemini API) also uses the deprecated model name. Updated to match the new model name to keep the existing diagnostic tool accurate.
📄 server/test_hybrid_ai.js
// testGeminiAPI() function - const model = genAI.getGenerativeModel({ model: 'gemini-1.5-flash', ... }) + const model = genAI.getGenerativeModel({ model: 'gemini-2.0-flash', ... })
